07 INDICES

What are Indices

Ein Oracle Server Index ist ein schema, mit dem man die Suche nach Zeilen bessern kann, indem man einen Pointer verwendet. Indizes können manuell oder automatisch erstellt werden. Wenn man keinen Index verwendet, muss jedes mal die ganze Tabelle durchsucht werden, um einen Wert zu finden.

Eine ROWID ist ein base64 string von der Adresse.

Indizes können jederzeit gelöscht werden, ohne dass es zu Problemen kommt.

Note: Wenn man eine Tabelle löscht, werden auch die Indizes gelöscht.

Types of Indices

Two types of Indices can be created

  1. Unique index: Der Oracle Server erstellt automatisch diesen Index wenn man einer Spalte in einer Tabelle den PRIMARY KEY oder den UNIQUE KEY zuweist. Der Name des Indexes ist dann der Name des Contraints.
  2. Nonunique index: Das ist ein Index, den der User hinzufügen kann, um den Zugriff auf die Zeilen zu verbessern. Zum Beispiel um Joins zu optimiernen, kann man einen Index auf die FOREIGN KEY Spalte legen.

Creating an Index

Man erstellt einen index auf eine oder mehrere Spalten mit dem CREATE INDEX Statement: 

CREATE INDEX index_name
ON table_name(column...,column)

Damit man einen Index erstellen kann, muss man die Rechte besitzen, eine Tabelle zu erstellen.
Weiters benötigt man auch die CREATE ANY INDEX Rechte.

When to create an Index

An Index should only be created, if:

  • Die Spalte sehr viele Daten enthält.
  • Eine Spalte eine große Menge an null values enthält.
  • Eine oder mehrere Spalten öfter zusammen in der WHERE Klausel oder einem JOIN verwendet werden.
  • Die Tabelle groß ist, und man bei den Abfragen normalerweise nur 2-4% der Zeilen ausgibt.

When not to create an Index

Jede DML operation (INSERT, UPDATE, DELETE) die auf der Tabelle ausgeführt wird, sorgt dafür, dass die Indexe upgedated werden müssen.

It is usually not worth creating an index, if:

  • Es eine kleine Tabelle ist.
  • Die Spalte nicht sehr oft in einer Abfrage verwendet werden
  • Die meisten Abfragen mehr als 2-4% der Zeilen ausgeben.
  • An der Tabelle oft DML operationen ausgeführt werden.

Composite Index

Ein composite index („concatenated“ index„ ist ein index, der auf mehrere Spalten einer Tabelle verweist. Composite indexes verbessern das lesen der Daten, dieser combination von Spalten.

NULL values werden ignoriert. Wenn die WHERE Klausen ein IS NULL enthält, wird kein Index verwendet.

Confirming Indexes

Man kann im data dictionary USER_INDEXES nachsehen, ob ein Index existiert. Weiters ist es auch möglich über die View USER_IND_COLUMNS die Existenz von Indexes zu beweisen.

Function-based Index

Ein funtion-based Index speichert die indexierten Daten und verwendet den Index basieren auf einem Select-Statement um daten zurückzugeben.

Function-based Indexes sind hilfreich, wenn man nicht weis, in welcher Form die Daten abgespeichert wurden.

Beispiel:

CREATE INDEX upper_last_name_idx
ON employees (UPPER(last_name));

SELECT * 
FROM employees
WHERE UPPER(last_name) = ’KING’;

Function-based indexes mit UPPER oder LOWER erlauben case-insensitive Suche.

Damit man sicher gehen kann, dass der Oracle Server den Index verwendet, anstatt einen komplette Scan zu machen, muss man sicher gehen, dass der Wert der Funktion in Subselects nicht null ist. Zum Beispiel verwendet das folgende Statement einen Index. Ohne der WHERE Klausel würde es einen full scan machen. 

SELECT *
FROM d_partners
WHERE UPPER (last_name) IS NOT NULL
ORDER BY UPPER (last_name);

Jede Oracle Built-in Funktion kann verwendet werden. Man kann auch Database Functions selbst schreiben. Wenn man eigene Funktionen schreibt, die einen function-based Index verwenden sollen, muss man das Keyword DETERMINISTIC im header verwenden.

In Oracle, das Deterministic Keyword bestimmt, dass wenn man die funktion immer mit den gleichen Daten aufruft, auch immer die gleichen Daten wieder heraus kommen.

Die Funktionen UPPER, LOWER oder TO_CHAR sind zum Beispiel funktionen die als deterministic definiert sind.

Removing an Index

Man kann keine Indexes verändern. Man muss ihn zuerst löschen und dannn neu erstellen-

Mit DROP INDEX name wird er gelöscht. Views und Sequences bleiben erhalten. 

DROP INDEX upper_last_name_idx;
