08 HIERARCHICAL SQL

What are hierarchies?

Hierarchien sind Baum-ähnliche strukturen, die dir helfen können, die Struktur deiner Daten zu verstehen.
Es gibt zwei Arten von Hierachien:

  • mono-hierarchy
  • polyhierarchy

In einer mono-hierarchy hat jedes Element maximal ein Element über sich, wähernd es in der poly-hierarch möglich ist, dass jedes Element mehrere Element über sich hat. Früher wurden rekursive Tasks mit stored Procedures/Functions gelöst. Rekursives und hierarchisches SQL gibt es seit 1999 und dadurch konnte man solche Probleme einfach mit SQL lösen.

Beispiel: Jeder Mitarbeiter hat einen Manager über sich (außer der Chef). Das wäre eine mono-hierarchy.

Die Mitarbeiterdaten würden so aussehen:

Parent and Child Values

Jeder Wert in einer hierarchy hat einen Parent value (außer der King). Die Werte direkt unter einem Parent werden Children genannt.

Siblings

Jeder Parent kann mehrere Children haben. Werte, die die gleichen Parents haben, werden Siblings genannt.

Descendant and ancestor values

Jedes Child kann auch Children haben. Mehrere Level die schlussendlich von einem Parent abhängen werden Descendant and ancestor values genannt.

Leaf Values

Leaf Values sind Elemente, die keine Children haben. Die Daten von non-leaf Elementen beziehen sich auf deren Children.

Die wichtigsten Klauseln sind START WITH und CONNECT BY:

  • Die Root einer Hierarchie bekommt man mit der START WITH Klausel
  • Die Verbindung der Zeilen ist mit CONNECT BY festgelegt.

Beispiel:
<codesql> [START WITH condition] CONNECT BY [NOCYCLE] condition </code>

Pseudocolumns

Name Description
LEVEL Gibt das level von der Root weg zurück. Die root selbst hat das Level 1
CONNECT_BY_ISCYCLE Gibt 1 zurück wenn das aktuelle Element ein Child hat, das auch sein Vorfahre ist. Man muss es in Verbindung mit NOCYCLE und CONNECT BY verwenden.
CONNECT_BY_ISLEAF Gibt 1 zurück, wenn as aktuelle Element ein leaf ist. Sonst gibt es 0 zurück.

Functions

Name Description
SYS_CONNECT_PATH(column,char) Gibt den Pfad des Elements von der Root node zurück. Die Elemente werden mit dem char getrennt.

Operators

Name Description
SYS_CONNECT_BY_ROOT Der Wert der Spalte der Root-zeile wird zurück gegeben
PRIOR Es wird verwendent, um Parent nodes zu verbindnen. Es sorgt dafür, dass der Wert der Parten node in der Child node verwendet wird.
Wird immer mit CONNECT BY verwendet.

Clauses

Name Description
START WITH condition Bestimmt die Root-Zeile nach der Bedingung
CONNECT BY … PRIOR Es spezifiziert die Beziehung zwischen Parten und Child nodes. Der PRIOR Operator bezieht sich auf die Parent row.
NOCYCLE verhindert Schleifen: Mitarbeiter A hat Mitarbeiter B als Vorgesetzten und Mitarbeiter B hat Mitarbeiter A als Vorgesetzten. Dies würde eine Schleife ergeben. Mit dem Command CONNECT_BY_ISCYCLE kann man sehen wo loops sind.
ORDER SIBLINGS BY Sortiert die Zeilen nach den Siblings der gleichen Parents

Processing of hierarchical queries

  • Ein Join wird zuerst ausgeführt
  • Die CONNECT BY condition wird ausgeführt.
  • Die überbleibende WHERE Klausel wird ausgeführt.
Formation of the hierarchy using the information from these evaluations
  1. Oracle holt die Root Nodes. (Diese mit der START WITH Bedingung)
  2. Oracle holt die Child Nodes von der Root Node. Die CONNECT BY condition muss mit einer der Root Nodes zusammenpassen.
  3. Oracle holt sich die Children der Zeilen aus dem Schritt zuvor. Und immer so weiter.
  4. Wenn kein Join verwendet wird, werden alle Zeilen, die nicht den Bedingungen der WHERE Klausel entsprechen gelöscht.
