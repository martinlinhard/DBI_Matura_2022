Graphendatenbank

Eine Graphendatenbank ist eine Datenbank, die Graphen benutzt um vernetzte Informationen darzustellen und abzuspeichern. Sie bieten meist spezialisierte Graphenalgorithem, um Abfragen zu ermöglichen, die mit relationalen Datenbanken nur sehr kompliziert umsetzbar sind. Beispiele:

  • Direkte und indirekte Nachbarn finden
  • Den kürzesten Pfad zwischen zwei Knoten berechnen
  • Stark und schwach vernetzte Knoten finden

Einsatz

Graphendatenbanken kommen vorallem bei vernetzten Daten zum Einsatz.

Anwendungsbeispiele

  • Vorschläge (von Produkten, Suchanfragen, Content,…)
  • Social Media und Social Networks
  • Management von Berechtigungen
  • Darstellen und verwalten von Netzwerk-Infrastruktur
  • Analyse von Datenbeziehungen (Zum Beispiel für Fraud Detection)

Neo4j

Neo4j ist eine in Java implementierte Graphendatenbank und folgt dem ACID Prinzip. In Neo4j wird alles entweder als Kante, als Knoten oder als Attribut gespeichert. Jeder Knoten hat eine beliebige Anzahl von Attributen. Knoten und Kanten können eine Beschriftung (Label) tragen. Beschriftungen können verwendet werden, um die Treffermenge bei Suchen einzuschränken. Die Abfrage der Daten wird über die Query-Language Cypher ermöglicht.

Beispiele

Erstellen eines Knoten

Wenn ein Knoten erstellt werden soll, ist es nicht unbedingt notwendig, einen Namen für den Knoten (= VarName in der Tabelle unten) zu vergeben.

Syntax
CREATE ( < VarName > : < Label > { < Attribut > : < Wert > })
Beispiel
CREATE ( p1 : Person { name : "Max" } )

Erstellen von mehreren Knoten und der dazugehörigen Beziehungen

Beim Erstellen von Beziehung können nur gerichtete Kanten (= Beziehungen) verwendet werden, also muss entweder links oder rechts von der Beziehung ein ← oder → sein.

Syntax
CREATE ( < VarName1 > : < Label > { < Attribut > : < Wert > }) ,
( < VarName2 > : < Label > { < Attribut > : < Wert > }) ,
( < VarName1 > ) - [ :< Beziehung > ] -> ( < VarName2 > )
// ODER
( < VarName1 > ) <- [ :< Beziehung > ] - ( < VarName2 > )
Beispiel
CREATE ( p1 : Person { name : "Max" } ),
( p2 : Person { name : "Maria" } ),
( p1 ) - [: wohnt_bei ] -> ( p2 ),
( p1 ) <- [: pflegt ] - ( p2 )

Erstellen von Beziehung zwischen bereits existierenden Knoten

Wenn Beziehung zwischen bereits existierenden Knoten erstellt werden sollen, müssen die Knoten zuerst mithilfe von MATCH herausgefunden und “gespeichert” werden, bevor eine Beziehung erstellt werden kann.

Syntax
MATCH ( < VarName1 > : < Label > { < Attribut > : < Wert > }) ,
( < VarName2 > : < Label > { < Attribut > : < Wert > }) ,
CREATE ( < VarName1 > ) - [ :< Beziehung > ] -> ( < VarName2 > )
Beispiel
MATCH ( p1 : Person { name : "Max" } ),
( p2 : Person { name : "Maria" } )
CREATE ( p1 ) - [: liebt ] -> ( p2 )

Löschen eines Knoten mit den dazugehörigen Beziehungen

Wenn ein Knoten gelöscht wird, der Beziehungen zu anderen Knoten hat, muss das Schlüsselwort DETACH angegeben werden (entspricht CASCADE CONSTRAINTS bei SQL). Ansonsten muss das Schlüsselwort nicht angegeben werden.

Syntax
MATCH ( < VarName > : < Label > { < Attribut > : < Wert > })
DETACH DELETE < VarName >
Beispiel
MATCH ( p : Person { name : "Max" } )
DETACH DELETE p

Abfragen eines Knoten mit einem bestimmten Attributwert

Um einen bestimmten Knoten zu erhalten, muss dieser Knoten zuerst mit MATCH herausgefunden und danach mit RETURN angezeigt werden. Wenn nur der Wert eines bestimmten Attributes ausgegeben werden soll, muss dieses Attribut angegeben werden (Achtung: Case-Sensitive → Es macht einen Unterschied, ob das Attribut Name oder name heißt).

Syntax
MATCH ( < VarName > : < Label > { < Attribut > : < Wert > })
RETURN < VarName >
// ODER
RETURN < VarName > . < AttributWert >
Beispiel
MATCH ( p : Person { name : "Max" } )
RETURN p
// ODER
RETURN p . name

Einschränken einer Abfrage mit WHERE

Wenn die MATCH -Abfrage mehrere Ergebnisse zurückliefert, können diese noch mit WHERE weiter eingegrenzt werden.

Syntax
MATCH ( < VarName > : < Label > { < Attribut > : < Wert > })
WHERE < VarName > . < Attribut > = < Vergleichswert >
RETURN < VarName >
Beispiel
MATCH ( p : Person )
WHERE p . name = "Max"
RETURN p

Suche nach Knoten mit bestimmten Beziehungen

Die MATCH-Klausel ist nicht (wie unten) auf zwei Knoten und eine Beziehung beschränkt, sondern kann beliebig viele Knoten und Beziehungen enthalten

Syntax
MATCH ( < VarName > : < Label > { < Attribut > : < Wert > })
- [ :< Beziehung > ] ->
( < VarName > : < Label > { < Attribut > : < Wert > })
RETURN < .. >
Beispiel
MATCH ( p1 : Person ) - [: pflegt ] -> ( p2 : Person )
RETURN p1 . name AS Pfleger , p2 . name AS Patient
MATCH ( p1 : Person ) - [: wohnt_bei ] ->
( p2 : Person ) - [: pflegt ] -> ( p1 )
RETURN p1 AS Patient

Bestimmte Anzahl an Kanten zwischen Knoten

Hinter der Beziehungsbezeichnung kann eine Anzahl angegeben werden, die vorgibt, wie viele Kanten zwischen zwei bestimmten Knoten sein dürfen. Die Zahl hinter * entspricht der Anzahl der Kanten zwischen den beiden Knoten

Syntax
MATCH ( < VarName > : < Label > { < Attribut > : < Wert > })
- [ :< Beziehung > * < Anzahl > ] ->
( < VarName > : < Label > { < Attribut > : < Wert > })
RETURN p2
Beispiel
// Genau ein Knoten dazwischen
MATCH ( p1 : Person { name : "Max" } ) - [: verwandt_mit *2 ] ->
( p2 : Person )
RETURN p2
// Genau 4 Knoten dazwischen
- [: verwandt_mit *5 ] ->
// Direkt oder maximal ein Knoten dazwischen
- [: verwandt_mit *1..2 ]
// Direkt oder maximal 10 Knoten dazwischen
- [: verwandt_mit *1..11 ] ->

Kürzester Pfad zwischen zwei Knoten

Um den kürzesten Pfad zwischen zwei Knoten herausfinden zu können, können die Funktion SHORTESTPATH (für Anzahl an Knoten) oder ALLSHORTESTPATHS (für Verlauf der Knoten) verwendet werden.

Syntax
// Kürzester Pfad zwischen zwei Knoten
MATCH p = SHORTESTPATH // oder ALLSHORTESTPATHS
(( < VarName > : < Label > { < Attribut > : < Wert > } )
- [ :< Beziehung > * ] ->
( < VarName > : < Label > { < Attribut > : < Wert > } ))
RETURN LENGTH ( p ) - 1
// ODER
RETURN EXTRACT ( xy in NODES ( p ) | xy < . Attribut > )
Beispiel
// Anzahl der Knoten zwischen zwei Knoten
MATCH p = SHORTESTPATH
(( p1 : Person { name : "Max" } )
- [: kennt * ] ->
( p2 : Person { name : "Maria" } ))
RETURN LENGTH ( p ) - 1 AS Anzahl
// Verlauf der Knoten zwischen zwei Knoten
MATCH p = ALLSHORTESTPATHS
(( p1 : Person { name : "Max" } )
- [: kennt * ] ->
( p2 : Person { name : "Maria" } ))
RETURN EXTRACT ( pers in NODES ( p ) | pers . name )
AS Verlauf

Österreich Beispiel

CREATE (b:Stadt{name:"Bregenz"}),
(g:Stadt{name:"Graz",country:"Steiermark", mayor:"Siegfried Nagl",
 population:282000}),
(i:Stadt{name:"Innsbruck",country:"Tirol", population:130894}),
(l:Stadt{name:"Linz",country:"Oberösterreich"}),
(s:Stadt{name:"Salzburg", mayor:"Harald Preuner"}),
(w:Stadt{name:"Wien",population:1867960}),
(donau:Fluss{name:"Donau"}),
(inn:Fluss{name:"Inn"}),
(mur:Fluss{name:"Mur"}),
(b)-[:direkt_nach]->(i),
(g)-[:direkt_nach]->(w),
(g)-[:direkt_nach]->(i),
(i)-[:direkt_nach]->(b),
(i)-[:direkt_nach]->(l),
(l)-[:direkt_nach]->(g),
(l)-[:direkt_nach]->(w),
(w)-[:direkt_nach]->(l),
(w)-[:direkt_nach]->(g),
(g)-[:liegt_an]->(mur),
(l)-[:liegt_an]->(donau),
(i)-[:liegt_an]->(inn),
(w)-[:liegt_an]->(donau);

//===============================================================================
// Welche Städte haben mehr als 200.000 Einwohner?
MATCH (s:Stadt)
WHERE s.population > 200000
RETURN s;

// Welche Städte haben mehr als 200.000 Einwohner?
// Ausgabe name
MATCH (s:Stadt)
WHERE s.population > 200000
RETURN s.name;

//===============================================================================
// Welche Städte haben weniger als 200.000 Einwohner?
MATCH (s:Stadt)
WHERE s.population < 200000
RETURN s;

//===============================================================================
// Welche Städte sind von Graz mit einem Direktflug erreichbar?
MATCH(s1:Stadt{name:"Graz"})-[:direkt_nach]->(s2:Stadt)
RETURN s2;

MATCH(s1:Stadt{name:"Linz"})-[:direkt_nach]->(s2:Stadt)
RETURN s2;

MATCH(s1:Stadt{name:"Graz"})-[:direkt_nach]->(s2:Stadt)
RETURN s2.name;

//===============================================================================
// Welche Städte an der Donau sind von Graz mit einem Direktflug erreichbar?
MATCH (s1:Stadt{name:"Graz"})-[:direkt_nach]->
(s2:Stadt)-[:liegt_an]->(f:Fluss{name:"Donau"})
RETURN s2;

//===============================================================================
// Für welche Direktflüge gibt es auch einen Direktflug in die Gegenrichtung
MATCH (s1:Stadt)-[r1:direkt_nach]->(s2:Stadt)-[r2:direkt_nach]->(s1)
RETURN s1.name +" nach "+ s2.name;

//===============================================================================
// Welche Städte sind von Graz mit genau einmal Umsteigen erreichbar?
MATCH(s1:Stadt{name:"Graz"})-[:direkt_nach]->
(s2:Stadt)-[:direkt_nach]->(s3:Stadt)
RETURN s3.name;

MATCH(s1:Stadt{name:"Graz"})-[:direkt_nach]->
(s2:Stadt)-[:direkt_nach]->(s3:Stadt)
WHERE NOT s3.name = "Graz"
RETURN DISTINCT s3.name;

MATCH(s1:Stadt{name:"Graz"})-[:direkt_nach*2]->(s2:Stadt)
WHERE NOT s2.name = "Graz"
RETURN DISTINCT s2.name;

//===============================================================================
// Welche Städte sind von Graz mit höchstens einmal umsteigen erreichbar?
MATCH(s1:Stadt {name:"Graz"}) -[:direkt_nach*0..2]-> (s2:Stadt)
WHERE NOT s2.name = "Graz"
RETURN s2;

MATCH(s1:Stadt {name:"Graz"}) -[:direkt_nach*0..2]-> (s2:Stadt)
WHERE NOT s2.name = "Graz"
RETURN s2.name;

MATCH(s1:Stadt {name:"Graz"}) -[:direkt_nach*0..2]-> (s2:Stadt)
WHERE NOT s2.name = "Graz"
RETURN DISTINCT s2.name;

MATCH(s1:Stadt {name:"Bregenz"}) -[:direkt_nach*0..2]-> (s2:Stadt)
WHERE NOT s2.name = "Bregenz"
RETURN DISTINCT s2.name;

//===============================================================================
// Wie oft muss man bei einer Reise von Bregenz nach Wien mindestens umsteigen?

MATCH p=SHORTESTPATH((s1:Stadt {name: "Bregenz"})
-[:direkt_nach*]->
(s2:Stadt{name:"Wien"})
)
RETURN LENGTH(p)-1;

MATCH (s1:Stadt {name: "Bregenz"}),
(s2:Stadt {name: "Wien"}),
p=SHORTESTPATH((s1)-[:direkt_nach*]->(s2))
RETURN LENGTH(p)-1;

//===============================================================================
// Wie ist der Reiseverlauf (Liste der Städte) bei einer Reise von Wien nach Bregenz?
MATCH p=SHORTESTPATH((s1:Stadt {name: "Wien"})
-[:direkt_nach*]->
(s2:Stadt{name:"Bregenz"})
)
RETURN EXTRACT(s in NODES(p) | s.name)

MATCH p=SHORTESTPATH((s1:Stadt {name: "Graz"})
-[:direkt_nach*]->
(s2:Stadt{name:"Linz"})
)
RETURN EXTRACT(s in NODES(p) | s.name)

MATCH p=ALLSHORTESTPATHS((s1:Stadt {name: "Graz"})
-[:direkt_nach*]->
(s2:Stadt{name:"Linz"})
)
RETURN EXTRACT(s in NODES(p) | s.name)
